# Русификация для чайнков

Это перепост знаменитого блога
[Си для чайников](https://c-for-dummies.com/blog/) от
[Дэна Гукина](https://uk.wikipedia.org/wiki/%D0%94%D0%B5%D0%BD_%D0%93%D0%BE%D0%BE%D0%BA%D1%96%D0%BD).
За основу взяты его последние работы
[github-репозитория](https://github.com/dangookin/C-For-Dummies-Blog),
а также в ручную перекладываемые и исправляемые исходные файлы на C
из блога между 2013 и 2019 годами.

Примеры разложены по годам каталогов `2013/` ... `2026/` .
В каталогах имена обрезаны до месаца и дня публикации.
Сохраняется и преемственность суффиксов, когда несколько примеров
в одном посте оканчиваются на `-a.c`, `-b.c` и т.д.
По аналогии, где возможно добавляются:

* `-r.c` - примеры с русскими буквами (чаще).
* `-s.c` - -//- на каком-то другом славянском языке (реже).

В каталоге `misc/` лежат фалы:

* `clean.sh` - очистка откомпилированных в Linux кодов примеров
* `L11N.sed` - шаблон для обработки исходных примеров на английском
  языке из функций и записи в формате
  [ASCII](https://ru.wikipedia.org/wiki/ASCII) в формат
  [широких символов](https://ru.wikipedia.org/wiki/%D0%A8%D0%B8%D1%80%D0%BE%D0%BA%D0%B8%D0%B9_%D1%81%D0%B8%D0%BC%D0%B2%D0%BE%D0%BB)
  `wchar_t`. Запуск: `sed -f L11N.sed исходный_файл_на.c > результат_для_локализации.с`.
  Замечание: после применения sed скрипта, полученный файл нужно
  доредактировать, исправить лишние дубли `setlocale()` внутри функций
  (кроме стартовой `main()`). После редактирования рекомендуется
  проверить изменения с помощью команды
  [diff](https://ru.wikipedia.org/wiki/Diff). Возможно, имеет смысл
  использовать команду [patch](https://ru.wikipedia.org/wiki/Patch_(Unix))
  вместо [sed](https://ru.wikipedia.org/wiki/Sed)?


Сборка производилась с использованием компиляторов
[gcc](https://ru.wikipedia.org/wiki/GNU_Compiler_Collection)
и [tcc](https://ru.wikipedia.org/wiki/Tiny_C_Compiler) в
Debian-подобной операционной системе (ОС). Задействовались стандартные
возможности библиотеки
[GNU LibC](https://ru.wikipedia.org/wiki/Glibc). Для сборки программ
применялось средство GNU make с инициализацией переменных `$CC`,
`$CFLAGS`, `$LDLIBS` и т.д.
